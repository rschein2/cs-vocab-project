Linux File System Hierarchy Flashcards

What's stored in /etc? What type of files go there? Where do you look for configuration files for system services like SSH or Apache?
/etc - System-wide configuration files

Why: "et cetera" - all config files for the system and installed software. Text files, human-editable.

What's in /etc:

  • /etc/passwd - User account information
  • /etc/shadow - Encrypted passwords
  • /etc/group - Group information
  • /etc/fstab - Filesystem mount configuration
  • /etc/hosts - Static hostname-to-IP mappings
  • /etc/hostname - System hostname
  • /etc/resolv.conf - DNS configuration
  • /etc/ssh/sshd_config - SSH server config
  • /etc/nginx/ - Nginx web server config
  • /etc/apt/ - APT package manager config (Debian/Ubuntu)
  • /etc/systemd/ - systemd configuration

Examples:

# Common tasks
sudo nano /etc/ssh/sshd_config  # Configure SSH
sudo nano /etc/fstab             # Configure auto-mount drives
cat /etc/os-release              # Check distro version
ls /etc/*.conf                   # List all .conf files
Best practice: Always backup config files before editing: sudo cp /etc/file /etc/file.bak

What's stored in /var? Where do you find system logs? What about mail, caches, and temporary application data?
/var - Variable data that changes during system operation

Why: "Variable" - files that grow and change: logs, databases, caches, spool files. Separate from root to prevent filling up critical filesystems.

What's in /var:

  • /var/log/ - Log files (syslog, auth.log, apache, nginx, etc.)
  • /var/cache/ - Application cache data
  • /var/tmp/ - Temporary files preserved between reboots
  • /var/spool/ - Spool directories (mail, print jobs, cron)
  • /var/mail/ - User mailboxes
  • /var/lib/ - State information for packages (databases, etc.)
  • /var/www/ - Web server document root (common location)

Examples:

# View system logs
sudo less /var/log/syslog         # System messages (Debian/Ubuntu)
sudo less /var/log/messages       # System messages (RHEL/Fedora)
sudo less /var/log/auth.log       # Authentication logs
sudo journalctl                   # systemd journal (modern alternative)

# Check disk usage (var can get large!)
du -sh /var/*

# Common log locations
/var/log/apache2/access.log  # Apache web logs
/var/log/nginx/access.log    # Nginx web logs
/var/log/mysql/error.log     # MySQL errors
/var/log/kern.log            # Kernel messages
/var vs /tmp: /var/tmp persists across reboots, /tmp is cleared

What's the difference between /bin, /sbin, /usr/bin, and /usr/sbin? Where do user commands live? What about system administration commands?
/bin - Essential user binaries (ls, cat, cp, bash)
/sbin - Essential system binaries (fsck, init, reboot)
/usr/bin - Non-essential user programs
/usr/sbin - Non-essential system programs

Why: Historical separation: /bin and /sbin are essential for boot and recovery. /usr/* are for normal operation.

What's where:

/bin (essential user commands):

  • ls, cp, mv, rm, cat, bash, sh, echo
  • Needed to boot and repair system

/sbin (essential system commands):

  • fsck, init, reboot, shutdown, ifconfig
  • Typically require root privileges

/usr/bin (user applications):

  • python, git, vim, gcc, make, wget
  • Most programs you install go here

/usr/sbin (system administration):

  • useradd, groupadd, sshd, apache2
  • System daemons and admin tools

Examples:

# Find which directory a command is in
which python3    # /usr/bin/python3
which ls         # /bin/ls
type -a python   # Show all locations

# Modern systems often symlink /bin â†’ /usr/bin
ls -l /bin  # May show: /bin -> usr/bin
Modern trend: Many distros now merge /bin into /usr/bin via symlinks for simplicity

What's stored in /home vs /root? Where are user files kept? What about the root user's files? What's in /usr vs /usr/local?
/home - Regular user home directories (/home/username)
/root - Root user's home directory
/usr - User programs (system-managed)
/usr/local - Locally installed software (admin-managed)

Why: Separation keeps user data safe from system changes and vice versa.

/home:

  • /home/alice/ - Alice's files
  • /home/bob/ - Bob's files
  • User configs: ~/.bashrc, ~/.ssh/, ~/.config/
  • Often on separate partition for easy backup

/root:

  • Root user's home (NOT /home/root!)
  • Root configs: /root/.bashrc, /root/.ssh/

/usr vs /usr/local:

  • /usr - Package manager installs here (apt, dnf, pacman)
  • /usr/local - You compile/install here manually
  • /usr/local/bin - Your custom scripts and programs

Examples:

# Go to your home directory
cd ~
cd $HOME
cd  # Just cd with no args

# Go to another user's home (if you have permission)
cd ~alice

# Install to /usr/local (custom software)
./configure --prefix=/usr/local
make
sudo make install

# Add /usr/local/bin to PATH
export PATH=/usr/local/bin:$PATH
Tilde expansion: ~ always means your home directory, regardless of where you are

What's /tmp used for? What about /opt? What's /dev, /proc, and /sys? Where do you find device files and kernel information?
/tmp - Temporary files (cleared on reboot)
/opt - Optional/third-party software packages
/dev - Device files (hardware interfaces)
/proc - Process and kernel information (virtual filesystem)
/sys - Kernel and hardware info (virtual filesystem)

/tmp (temporary files):

  • Cleared on reboot (or periodically)
  • World-writable but has sticky bit (users can only delete their own files)
  • Use for scratch space, download temp files

/opt (optional software):

  • Large third-party applications (Chrome, Slack, custom software)
  • Each app in its own directory: /opt/google/chrome/

/dev (device files):

  • /dev/sda, /dev/sdb - Hard drives
  • /dev/sda1 - First partition on first drive
  • /dev/null - Discard output (black hole)
  • /dev/zero - Infinite zeros
  • /dev/random, /dev/urandom - Random data
  • /dev/tty - Current terminal

/proc (process info - virtual):

  • /proc/cpuinfo - CPU information
  • /proc/meminfo - Memory information
  • /proc/PID/ - Info about process PID
  • /proc/sys/ - Kernel tunables

/sys (system/kernel - virtual):

  • Hardware and driver information
  • /sys/class/net/ - Network interfaces
  • /sys/block/ - Block devices

Examples:

# Use /tmp for scratch work
cp largefile.zip /tmp/
cd /tmp && unzip largefile.zip

# Discard output
command > /dev/null 2>&1

# Get random data
head -c 16 /dev/urandom | base64  # Random password

# Check CPU info
cat /proc/cpuinfo
lscpu  # Nicer output

# Check memory
cat /proc/meminfo
free -h  # Nicer output

# List disks and partitions
ls /dev/sd*
lsblk  # Better view

What's in /lib and /usr/lib? Where are shared libraries stored? What's /boot used for? What about /mnt and /media?
/lib - Essential shared libraries for /bin and /sbin
/usr/lib - Libraries for /usr/bin programs
/boot - Boot loader files (kernel, initramfs)
/mnt - Temporary mount point for administrators
/media - Auto-mounted removable media (USB, CD/DVD)

/lib and /usr/lib (libraries):

  • Shared libraries (.so files - like Windows DLLs)
  • /lib - Libraries for essential programs
  • /usr/lib - Libraries for user programs
  • /usr/local/lib - Libraries for locally compiled software

/boot (boot files):

  • Kernel images (vmlinuz-*)
  • Initial RAM disk (initrd.img-* or initramfs-*)
  • Boot loader config (grub/)
  • DO NOT fill this partition - system won't boot!

/mnt and /media (mount points):

  • /mnt - Manual mounts by admin (NFS shares, extra drives)
  • /media - Auto-mounted USB drives, CDs, etc.
  • /media/username/USBDrive/ - Typical USB mount location

Examples:

# Check shared library dependencies
ldd /bin/ls  # Shows which libraries ls needs

# Boot partition usage (don't let it fill up!)
df -h /boot

# Clean old kernels to free /boot space
sudo apt autoremove  # Debian/Ubuntu
sudo dnf remove $(dnf repoquery --installonly --latest-limit=-2)  # Fedora

# Mount drive manually
sudo mount /dev/sdb1 /mnt/external

# Check auto-mounted media
ls /media/$USER/

# Unmount
sudo umount /mnt/external
Library path: LD_LIBRARY_PATH environment variable controls where to search for libraries

How do you navigate the file system hierarchy quickly? What commands show where you are, where a program is located, or what type of file something is?
Current directory: pwd
Find program: which program or type program
File type: file filename
Find file: find / -name filename or locate filename

Examples:

# Where am I?
pwd  # Print working directory

# Find where a command is
which python3    # /usr/bin/python3
type ls          # ls is aliased to `ls --color=auto'
whereis python3  # Shows binary, source, and man page locations

# What is this file?
file /bin/ls           # ELF executable
file document.pdf      # PDF document
file image.jpg         # JPEG image data
file /dev/sda          # block special

# Find files
find /etc -name "*.conf"        # All .conf files in /etc
locate sshd_config              # Fast search (uses index)
sudo updatedb                   # Update locate database

# Navigate quickly
cd -        # Go back to previous directory
cd ~        # Go home
cd /        # Go to root
cd ../..    # Go up two levels

# Show directory tree
tree /etc/ssh           # Visual tree (install tree package)
ls -R /etc/ssh          # Recursive list

# Find recently modified files
find /etc -mtime -1     # Modified in last 24 hours
find /var/log -mmin -30 # Modified in last 30 minutes
Tip: Use tab completion heavily! Type first few letters and hit Tab to autocomplete paths
