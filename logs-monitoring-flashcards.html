Your system service is failing and you need to check recent logs. How do you view systemd logs? What if you want to follow logs in real-time?
View recent logs: journalctl -xe
Follow in real-time: journalctl -f

Why: journalctl queries systemd journal. -x adds explanatory help text, -e jumps to end. -f follows like tail -f.

Alternatives:

  • journalctl -n 50 (show last 50 lines)
  • journalctl --since "10 minutes ago" (time-based filter)
  • journalctl -b (logs since last boot)

You need to check logs for a specific systemd service. How do you view service-specific logs? What if you want only errors?
Logs for specific service: journalctl -u servicename
Only errors/warnings: journalctl -u servicename -p err

Why: -u filters by unit (service name). -p filters by priority: emerg, alert, crit, err, warning, notice, info, debug.

Alternatives:

  • journalctl -u nginx -f (follow nginx logs)
  • journalctl -u ssh --since today (ssh logs from today)
  • journalctl -p warning (all warnings+ system-wide)

You want to see system uptime and load average. How do you check uptime? What do the load average numbers mean?
Check uptime and load: uptime
Load meaning: 1-min, 5-min, 15-min averages of runnable processes

Why: uptime shows how long system is running + load averages. Load > CPU cores = system under pressure. Load < cores = system handling load fine.

Interpretation:

  • 4-core system: load of 4.0 = 100% utilized
  • load of 8.0 = 200% (queued processes)
  • load of 0.5 = 12.5% utilized
  • Also shown in top, w, cat /proc/loadavg

You need to monitor memory usage including cache and swap. How do you see detailed memory stats? What's the difference between used and available?
Memory stats: free -h
Detailed breakdown: cat /proc/meminfo

Why: free -h shows total, used, free, cache/buffer, available memory. "Available" is what's actually free for apps (includes reclaimable cache). Cache is good!

Understanding:

  • High cache/buffers = normal (Linux uses free RAM for cache)
  • "available" matters more than "free"
  • Swap usage = memory pressure (unless configured as zram)
  • vmstat for ongoing memory stats
