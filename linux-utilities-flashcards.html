How do you manage system services with systemd? Start a service? Enable it to start at boot? Check its status?
Commands:
  • systemctl start service - start now
  • systemctl stop service - stop now
  • systemctl restart service - restart
  • systemctl enable service - auto-start at boot
  • systemctl disable service - don't auto-start
  • systemctl status service - check current status

Why systemctl: Modern Linux uses systemd to manage services. These commands replace older init.d scripts.

Common patterns:

sudo systemctl start nginx           # Start web server now
sudo systemctl enable nginx          # Make it start at boot
sudo systemctl restart ssh           # Restart SSH after config change
systemctl status postgresql          # Is database running?
sudo systemctl enable --now docker   # Enable AND start in one command

List services:

systemctl list-units --type=service       # All loaded services
systemctl list-unit-files --type=service  # All available services
systemctl list-units --state=failed       # Failed services

Why enable vs start:

  • start - runs now, but won't survive reboot
  • enable - starts at boot, but doesn't start now
  • enable --now - both!

View logs for a service:

journalctl -u service-name -f

How do you view system logs? What about following logs in real-time? Filtering by service or time?
Command: journalctl

Why journalctl: Systemd's logging system. Replaces reading text files in /var/log for many services.

Common patterns:

journalctl                          # All logs (paged)
journalctl -f                       # Follow logs in real-time (like tail -f)
journalctl -u nginx                 # Logs for nginx service only
journalctl -u nginx -f              # Follow nginx logs
journalctl --since "1 hour ago"     # Last hour
journalctl --since "2024-01-15"     # Since specific date
journalctl --since today            # Today's logs
journalctl -p err                   # Only errors (and worse)
journalctl -k                       # Kernel messages only
journalctl -b                       # Since last boot

Priority levels:

  • -p emerg - System unusable
  • -p err - Errors
  • -p warning - Warnings
  • -p info - Info messages

Useful combinations:

journalctl -u ssh -f --since "5 minutes ago"
journalctl -p err -b                # All errors since boot
journalctl -xe                      # Last entries with explanations

Disk usage:

journalctl --disk-usage             # How much space logs use
sudo journalctl --vacuum-time=2weeks  # Keep only 2 weeks
sudo journalctl --vacuum-size=1G      # Keep only 1GB

How do you connect to WiFi from the command line? List available networks? Disconnect?
Command: nmcli (NetworkManager CLI)

Why nmcli: NetworkManager's command-line interface. Works on most modern Linux desktops.

Common operations:

nmcli device wifi list              # List available WiFi networks
nmcli device wifi connect "SSID" password "PASSWORD"
nmcli connection up "Network Name"  # Connect to saved network
nmcli connection down "Network Name" # Disconnect
nmcli device disconnect wlan0       # Disconnect WiFi adapter

View connection status:

nmcli                               # General status
nmcli connection show               # All saved connections
nmcli device status                 # Network devices status
nmcli connection show --active      # Active connections only

Show WiFi password:

nmcli connection show "Network" --show-secrets | grep psk

Delete saved network:

nmcli connection delete "Network Name"

Alternative (older systems):

iwctl                               # For iwd (iNet wireless daemon)
# or
wpa_supplicant                      # Lower-level WiFi tool

Enable/disable WiFi:

nmcli radio wifi off
nmcli radio wifi on

How do you adjust screen brightness? What about getting current brightness level? Set to a specific percentage?
Commands:
  • brightnessctl - modern, user-friendly
  • xrandr --output SCREEN --brightness 0.7 - X11 alternative

brightnessctl usage:

brightnessctl                       # Show current brightness
brightnessctl set 50%               # Set to 50%
brightnessctl set +10%              # Increase by 10%
brightnessctl set 10%-              # Decrease by 10%
brightnessctl set 500               # Set to absolute value
brightnessctl -l                    # List devices

Why brightnessctl: Works with modern backlight systems, doesn't require X11, user-friendly.

xrandr alternative:

xrandr --output eDP-1 --brightness 0.5   # 50% brightness
xrandr --output eDP-1 --brightness 1.0   # Full brightness (default)

Find your screen name:

xrandr | grep " connected"

Manual file method (advanced):

cat /sys/class/backlight/*/brightness      # Current value
cat /sys/class/backlight/*/max_brightness  # Max value
echo 500 | sudo tee /sys/class/backlight/*/brightness  # Set value

Keyboard shortcuts: Often Fn + F5/F6 or similar, handled by desktop environment.

How do you manage Bluetooth devices? Connect to a device? List paired devices? Turn Bluetooth on/off?
Command: bluetoothctl

Why bluetoothctl: Interactive Bluetooth management tool that works across different Linux systems.

Basic workflow:

bluetoothctl                        # Start interactive mode
power on                            # Turn on Bluetooth
scan on                             # Start scanning for devices
# Wait for device to appear...
pair AA:BB:CC:DD:EE:FF              # Pair with device
connect AA:BB:CC:DD:EE:FF           # Connect to device
trust AA:BB:CC:DD:EE:FF             # Auto-connect in future
exit                                # Leave bluetoothctl

Non-interactive commands:

bluetoothctl power on
bluetoothctl devices                # List known devices
bluetoothctl paired-devices         # List paired devices
bluetoothctl connect AA:BB:CC:DD:EE:FF
bluetoothctl disconnect AA:BB:CC:DD:EE:FF

Remove/unpair device:

bluetoothctl remove AA:BB:CC:DD:EE:FF

Show Bluetooth status:

bluetoothctl show                   # Controller info
rfkill list bluetooth               # Is Bluetooth blocked?

Unblock if disabled:

rfkill unblock bluetooth

How do you control audio volume? Mute/unmute? Switch audio output devices?
Commands:
  • pactl - PulseAudio control (modern)
  • amixer - ALSA mixer (older/simpler)

pactl (PulseAudio) - recommended:

pactl set-sink-volume @DEFAULT_SINK@ +5%    # Increase 5%
pactl set-sink-volume @DEFAULT_SINK@ -5%    # Decrease 5%
pactl set-sink-volume @DEFAULT_SINK@ 50%    # Set to 50%
pactl set-sink-mute @DEFAULT_SINK@ toggle   # Toggle mute
pactl list sinks                            # List audio outputs
pactl list sources                          # List audio inputs (mics)

Set specific output device:

pactl list sinks short              # Get sink ID/name
pactl set-default-sink SINK_NAME    # Set as default output

amixer (ALSA) alternative:

amixer set Master 5%+               # Increase volume
amixer set Master 5%-               # Decrease volume
amixer set Master 50%               # Set to 50%
amixer set Master toggle            # Toggle mute
amixer sget Master                  # Get current volume

Interactive TUI:

alsamixer                           # ncurses interface
# Use arrow keys, M to mute, Esc to exit

Why PulseAudio vs ALSA: PulseAudio sits on top of ALSA, provides per-application volume, network audio, etc.

How do you manage displays and monitors? Change resolution? Set up multiple monitors? Rotate a screen?
Command: xrandr (X11) or wlr-randr (Wayland)

Basic xrandr usage:

xrandr                              # List all displays and resolutions
xrandr --output HDMI-1 --mode 1920x1080   # Set resolution
xrandr --output HDMI-1 --rotate left      # Rotate screen
xrandr --output HDMI-1 --off              # Turn off display
xrandr --output HDMI-1 --auto             # Auto-detect best mode

Multiple monitor setup:

# External monitor to the right of laptop screen
xrandr --output eDP-1 --primary --output HDMI-1 --right-of eDP-1 --auto

# Mirror displays
xrandr --output HDMI-1 --same-as eDP-1

# External only, disable laptop screen
xrandr --output eDP-1 --off --output HDMI-1 --auto

Find display names:

xrandr | grep " connected"

Set refresh rate:

xrandr --output HDMI-1 --mode 1920x1080 --rate 144

Wayland alternative:

wlr-randr                           # List outputs
wlr-randr --output HDMI-A-1 --mode 1920x1080@60

GUI tools:

  • arandr - Graphical xrandr frontend
  • Settings app in your desktop environment

How do you list block devices and partitions? Mount a USB drive? Unmount safely?
List devices: lsblk

Why lsblk is useful: Shows all block devices (drives, partitions) in a tree format with size, mount points.

Example output:

NAME   MAJ:MIN RM  SIZE RO TYPE MOUNTPOINT
sda      8:0    0  500G  0 disk
├─sda1   8:1    0  512M  0 part /boot
└─sda2   8:2    0  499G  0 part /
sdb      8:16   1   32G  0 disk
└─sdb1   8:17   1   32G  0 part /media/usb

Mount USB drive:

lsblk                               # Find device (e.g., sdb1)
sudo mkdir -p /mnt/usb              # Create mount point
sudo mount /dev/sdb1 /mnt/usb       # Mount
cd /mnt/usb                         # Access files

Unmount safely:

sudo umount /mnt/usb                # Unmount
# or
sudo umount /dev/sdb1               # By device name

Why safe unmounting matters: Ensures all data is written to disk before removal. Otherwise, risk data corruption.

If device is busy:

lsof /mnt/usb                       # See what's using it
sudo umount -l /mnt/usb             # Lazy unmount (waits until free)

Auto-mount at boot (edit /etc/fstab):

/dev/sdb1  /mnt/usb  ext4  defaults  0  2

How do you set up a cron job? Edit crontab? What about different schedules (hourly, daily, weekly)?
Edit crontab: crontab -e

Why cron: Schedule commands to run automatically at specific times or intervals.

Crontab syntax:

# Min Hour Day Month Weekday Command
  *    *    *    *      *     /path/to/command

# Examples:
0  2  *  *  *  /backup.sh              # Daily at 2 AM
*/5  *  *  *  *  /check.sh              # Every 5 minutes
0  */6  *  *  *  /update.sh             # Every 6 hours
0  0  *  *  0  /weekly.sh               # Sundays at midnight
0  0  1  *  *  /monthly.sh              # 1st of month at midnight

Manage crontabs:

crontab -e                          # Edit your crontab
crontab -l                          # List current crontab
crontab -r                          # Remove crontab (careful!)
sudo crontab -u username -e         # Edit another user's crontab

Shorthand schedules:

@reboot    /script.sh               # Run at startup
@hourly    /script.sh               # Once per hour (0 * * * *)
@daily     /script.sh               # Once per day (0 0 * * *)
@weekly    /script.sh               # Once per week (0 0 * * 0)
@monthly   /script.sh               # Once per month (0 0 1 * *)

View cron logs:

grep CRON /var/log/syslog           # Debian/Ubuntu
journalctl -u cron                  # Systemd systems

Debug cron jobs: Redirect output to a log file:

0 2 * * * /script.sh >> /var/log/script.log 2>&1

How do you create a new user? Set their password? Add them to a group? Delete a user?
Create user: sudo useradd username

Better way (with home directory):

sudo useradd -m -s /bin/bash username
# -m creates home directory
# -s sets default shell

Set password:

sudo passwd username

Create user with one command (interactive):

sudo adduser username               # Debian/Ubuntu (user-friendly)
# Prompts for password, full name, etc.

Add user to groups:

sudo usermod -aG sudo username      # Add to sudo group (grant admin)
sudo usermod -aG docker username    # Add to docker group
sudo usermod -aG wheel username     # Add to wheel group (RHEL/Fedora admin)

Why -aG:

  • -a = append (don't remove from other groups)
  • -G = supplementary groups

Delete user:

sudo userdel username               # Keep home directory
sudo userdel -r username            # Delete with home directory

List users:

cat /etc/passwd                     # All users (system + human)
cut -d: -f1 /etc/passwd | sort      # Just usernames

Change user password:

passwd                              # Change your own password

How do you manage the firewall? Open a port? Check firewall status? What about different firewall tools?
Commands by tool:

UFW (Uncomplicated Firewall - Ubuntu/Debian):

sudo ufw status                     # Check status
sudo ufw enable                     # Enable firewall
sudo ufw allow 22                   # Open port 22 (SSH)
sudo ufw allow 80/tcp               # Open port 80 TCP only
sudo ufw allow ssh                  # Open SSH (by service name)
sudo ufw deny 23                    # Block port 23
sudo ufw delete allow 80            # Remove rule
sudo ufw status numbered            # Show rules with numbers
sudo ufw delete 3                   # Delete rule by number

firewalld (Fedora/RHEL/CentOS):

sudo firewall-cmd --state           # Check if running
sudo firewall-cmd --list-all        # Show all rules
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --reload          # Apply permanent changes
sudo firewall-cmd --add-service=http --permanent
sudo firewall-cmd --remove-port=80/tcp --permanent

iptables (low-level, all distros):

sudo iptables -L                    # List rules
sudo iptables -A INPUT -p tcp --dport 80 -j ACCEPT
# More complex, usually use ufw/firewalld instead

Common ports:

  • 22 - SSH
  • 80 - HTTP
  • 443 - HTTPS
  • 3000 - Many dev servers
  • 5432 - PostgreSQL

How do you check network connectivity? Test if a host is reachable? Trace the route to a host? See open ports?
Basic connectivity: ping hostname

Common network diagnostics:

ping google.com                     # Test connectivity
ping -c 4 google.com                # Send only 4 packets
traceroute google.com               # Trace route to host
tracepath google.com                # Alternative (no root needed)
mtr google.com                      # Continuous traceroute + ping

Check open ports and connections:

ss -tuln                            # All listening TCP/UDP ports (modern)
netstat -tuln                       # Same thing (older command)
ss -tunap                           # Include process names (-p)
lsof -i :80                         # What's using port 80?

Flags explained:

  • -t = TCP
  • -u = UDP
  • -l = listening
  • -n = numeric (don't resolve names)
  • -a = all connections
  • -p = process

DNS lookup:

nslookup google.com                 # Basic DNS query
dig google.com                      # Detailed DNS info
host google.com                     # Simple DNS lookup

Check local network configuration:

ip addr                             # IP addresses (modern)
ip route                            # Routing table
ifconfig                            # Network interfaces (older)

How do you set the system time and date? Configure timezone? Use network time sync (NTP)?
Command: timedatectl (systemd)

View current time/date settings:

timedatectl                         # Show current settings
date                                # Just show current date/time

Set timezone:

timedatectl list-timezones          # List all timezones
timedatectl set-timezone America/New_York
timedatectl set-timezone Europe/London
timedatectl set-timezone UTC

Enable NTP (network time sync):

timedatectl set-ntp true            # Enable automatic time sync
systemctl status systemd-timesyncd  # Check sync service status

Manually set date/time (if NTP disabled):

sudo timedatectl set-time "2024-01-15 14:30:00"
sudo timedatectl set-time "14:30:00"  # Time only

Why NTP is recommended: Keeps your clock accurate automatically. Important for logs, certificates, scheduled tasks.

Old way (non-systemd):

sudo date -s "2024-01-15 14:30:00"  # Set date/time
sudo hwclock --systohc              # Sync to hardware clock
ntpdate pool.ntp.org                # Sync with NTP server

Show hardware clock:

sudo hwclock --show

How do you manage clipboard content? Copy text to clipboard from terminal? Paste clipboard contents?
Commands:
  • xclip - X11 clipboard
  • wl-copy / wl-paste - Wayland clipboard

xclip (X11) usage:

echo "text" | xclip                 # Copy to primary selection
echo "text" | xclip -selection clipboard  # Copy to clipboard (Ctrl+V)
xclip -o                            # Paste from primary
xclip -selection clipboard -o       # Paste from clipboard
cat file.txt | xclip -selection clipboard  # Copy file to clipboard

Aliases for convenience:

alias pbcopy='xclip -selection clipboard'
alias pbpaste='xclip -selection clipboard -o'
# Then use:
echo "hi" | pbcopy
pbpaste

Wayland alternative:

echo "text" | wl-copy              # Copy to clipboard
wl-paste                            # Paste from clipboard
cat file.txt | wl-copy              # Copy file contents

Primary vs Clipboard selection:

  • Primary - middle-click paste (text you select)
  • Clipboard - Ctrl+C / Ctrl+V (explicit copy)

Copy command output:

ls -la | xclip -selection clipboard
pwd | pbcopy                        # Copy current directory

View clipboard manager:

clipmenu                            # If installed (clipboard history)

How do you check which kernel modules are loaded? Load a module? Unload a module? What about persistent loading?
Commands:

List loaded modules:

lsmod                               # List all loaded modules
lsmod | grep wifi                   # Search for specific module

Load a module:

sudo modprobe module-name           # Load module
sudo modprobe -r module-name        # Remove/unload module
sudo modprobe -v module-name        # Verbose (see what happens)

Why modprobe vs insmod: modprobe handles dependencies automatically. Always use modprobe unless you know what you're doing.

Module information:

modinfo module-name                 # Show module details
modinfo iwlwifi                     # WiFi driver info

Load module at boot:

# Add to /etc/modules or /etc/modules-load.d/module.conf
echo "module-name" | sudo tee -a /etc/modules

Blacklist a module (prevent loading):

# Create file /etc/modprobe.d/blacklist.conf
echo "blacklist module-name" | sudo tee -a /etc/modprobe.d/blacklist.conf
sudo update-initramfs -u            # Rebuild initramfs

Common modules:

  • bluetooth - Bluetooth support
  • snd_* - Sound drivers
  • iwlwifi - Intel WiFi
  • nouveau - NVIDIA open driver

Command: localectl (systemd)

localectl                           # Show current settings

# Edit /etc/locale.gen, uncomment desired locales

LANG=es_ES.UTF-8 command            # Run command with Spanish locale

Why UTF-8 is important: Supports all Unicode characters (emoji, international text, etc.). Always use UTF-8 unless you have specific reasons not to.

How do you manage the hostname? Change it temporarily? Permanently? What about pretty hostname?
Command: hostnamectl (systemd)

View hostname:

hostname                            # Show current hostname
hostnamectl                         # Detailed hostname info

Change hostname:

sudo hostnamectl set-hostname new-hostname
sudo hostnamectl set-hostname "My Computer" --pretty  # Pretty name
sudo hostnamectl set-hostname myserver --static       # Static name

Three types of hostname:

  • Static - permanent hostname (saved in /etc/hostname)
  • Pretty - human-friendly name with spaces
  • Transient - temporary, from DHCP/network

Temporary change (until reboot):

sudo hostname temp-name

Manual method (non-systemd):

echo "new-hostname" | sudo tee /etc/hostname
sudo hostnamectl set-hostname new-hostname
# or edit /etc/hostname and reboot

Update /etc/hosts:

sudo nano /etc/hosts
# Change:
127.0.0.1   localhost
127.0.1.1   new-hostname

Why change hostname: Identify machines on network, better than "ubuntu-desktop" or random string.

How do you manage swap space? Create a swap file? Enable/disable swap? Check swap usage?
Check swap:

View swap usage:

swapon --show                       # Show swap devices
free -h                             # Shows swap in memory overview
cat /proc/swaps                     # Swap file/partition details

Create swap file:

sudo fallocate -l 2G /swapfile      # Create 2GB file (fast)
# or
sudo dd if=/dev/zero of=/swapfile bs=1G count=2  # Alternative

sudo chmod 600 /swapfile            # Set permissions
sudo mkswap /swapfile               # Format as swap
sudo swapon /swapfile               # Enable swap

Make swap permanent (add to /etc/fstab):

echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

Enable/disable swap:

sudo swapon /swapfile               # Enable
sudo swapoff /swapfile              # Disable
sudo swapoff -a                     # Disable all swap
sudo swapon -a                      # Enable all swap in fstab

Remove swap file:

sudo swapoff /swapfile
sudo rm /swapfile
# Remove from /etc/fstab

Adjust swappiness (how aggressively to use swap):

cat /proc/sys/vm/swappiness         # Current value (default 60)
sudo sysctl vm.swappiness=10        # Less aggressive (prefer RAM)
# Make permanent: add to /etc/sysctl.conf
vm.swappiness=10

Why swap matters: Prevents OOM (out of memory) kills, enables hibernation, but slower than RAM.

How do you enable/disable services from starting at boot? What about socket activation? Mask a service?
Boot management:

Enable/disable auto-start:

sudo systemctl enable service       # Start at boot
sudo systemctl disable service      # Don't start at boot
sudo systemctl is-enabled service   # Check if enabled

Why enable doesn't start now: Separation of concerns. enable = boot behavior, start = right now.

Mask a service (prevent starting):

sudo systemctl mask service         # Cannot be started (symlinked to /dev/null)
sudo systemctl unmask service       # Undo masking

Why mask vs disable: disable prevents auto-start, but you can still manually start. mask prevents all starting.

Socket activation:

sudo systemctl enable sshd.socket   # Enable SSH socket
sudo systemctl start sshd.socket    # Start listening
# Service only starts when connection arrives (on-demand)

List enabled services:

systemctl list-unit-files --state=enabled
systemctl list-unit-files --type=service --state=enabled

Show service file location:

systemctl cat service               # Show service file contents
systemctl show service              # Show all properties

Reload after editing service file:

sudo systemctl daemon-reload        # Reload systemd configuration
sudo systemctl restart service      # Apply changes
