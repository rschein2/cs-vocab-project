You plugged in a USB drive and want to see all available storage devices. How do you list all block devices? What information does it show?
List block devices: lsblk
More details: lsblk -f

Why: lsblk shows all disks, partitions, and mount points in tree format. -f adds filesystem type, UUID, and mount options. Essential for disk management.

Alternatives:

  • lsblk -o NAME,SIZE,TYPE,MOUNTPOINT (custom columns)
  • sudo fdisk -l (more detailed partition info)
  • ls -l /dev/sd* (list disk devices directly)

You need to check filesystem usage and available space. How do you see disk space? What if you want to see inode usage?
Check disk space: df -h
Check inodes: df -i

Why: df -h shows filesystem space usage (human-readable). -i shows inode usage - can run out of inodes even with free space!

Alternatives:

  • df -h /path (check specific filesystem)
  • df -Th (include filesystem type)
  • df -h --total (add total row)

You need to mount a USB drive or external disk. How do you mount a filesystem? What if you want to mount to a specific location?
Mount disk: sudo mount /dev/sdb1 /mnt
Custom mount point: sudo mount /dev/sdb1 /media/usb

Why: mount attaches filesystem to directory tree. Device name (like /dev/sdb1) to mount point (/mnt). Directory must exist first.

Alternatives:

  • sudo mount -t ext4 /dev/sdb1 /mnt (specify filesystem type)
  • sudo mount -o ro /dev/sdb1 /mnt (read-only mount)
  • mount (no args: list all mounted filesystems)

You need to safely unmount a device before removing it. How do you unmount? What if the device is busy?
Unmount: sudo umount /mnt or sudo umount /dev/sdb1
If busy, find what's using it: sudo lsof +D /mnt

Why: umount detaches filesystem. Can specify mount point or device. "Device busy" means files are open or processes are using it.

Alternatives:

  • sudo fuser -m /mnt (find processes using mount point)
  • sudo umount -l /mnt (lazy unmount: detach when no longer busy)
  • sudo umount -f /mnt (force unmount - risky for NFS)

You want disks to mount automatically on boot. Where do you configure automatic mounting? What's the file format?
Configuration file: /etc/fstab
Format: UUID=xxx /mount/point filesystem options dump pass

Why: /etc/fstab (filesystem table) defines automatic mounts. Use UUID instead of /dev/sdX (device names can change). Wrong fstab can prevent boot!

Example entry:

  • UUID=12345 /data ext4 defaults 0 2
  • sudo blkid (get UUIDs)
  • sudo mount -a (test fstab without reboot)
  • Always test before rebooting!

You need to create partitions on a new disk. How do you partition a disk? What's the difference between MBR and GPT?
Partition with fdisk (MBR): sudo fdisk /dev/sdb
Partition with gdisk (GPT): sudo gdisk /dev/sdb

Why: fdisk for MBR (legacy, max 2TB, 4 primary partitions). gdisk for GPT (modern, large disks, many partitions). Interactive menu-driven.

Alternatives:

  • sudo parted /dev/sdb (supports both MBR and GPT)
  • sudo cfdisk /dev/sdb (user-friendly ncurses interface)
  • GPT recommended for new disks > 2TB

You created a partition and need to format it with a filesystem. How do you create an ext4 filesystem? What about other filesystem types?
Format as ext4: sudo mkfs.ext4 /dev/sdb1
Format as XFS: sudo mkfs.xfs /dev/sdb1

Why: mkfs (make filesystem) creates filesystem on partition. ext4 is default for Linux. XFS good for large files. This ERASES all data!

Alternatives:

  • sudo mkfs.ext4 -L DataDisk /dev/sdb1 (add filesystem label)
  • sudo mkfs.vfat /dev/sdb1 (FAT32, for Windows compatibility)
  • sudo mkfs.ntfs /dev/sdb1 (NTFS, for Windows)
  • sudo mkfs.btrfs /dev/sdb1 (Btrfs, advanced features)

You want to check a filesystem for errors. How do you run a filesystem check? When can you check a filesystem?
Check filesystem: sudo fsck /dev/sdb1
Check ext4 specifically: sudo fsck.ext4 /dev/sdb1

Why: fsck (filesystem check) repairs errors. MUST be unmounted first or system won't boot! Can run automatically on boot if configured.

Alternatives:

  • sudo fsck -y /dev/sdb1 (auto-answer yes to repairs)
  • sudo e2fsck -f /dev/sdb1 (force check even if clean)
  • Check at boot: tune2fs -c 30 /dev/sdb1 (every 30 mounts)

You want to see filesystem properties and tune them. How do you view ext4 filesystem info? How do you change settings?
View filesystem info: sudo tune2fs -l /dev/sdb1
Set filesystem label: sudo tune2fs -L "MyData" /dev/sdb1

Why: tune2fs views/modifies ext2/ext3/ext4 filesystems. -l lists all parameters. Can change label, reserved space, check intervals.

Alternatives:

  • sudo tune2fs -m 1 /dev/sdb1 (set 1% reserved space for root)
  • sudo tune2fs -c 0 /dev/sdb1 (disable mount count check)
  • sudo blkid /dev/sdb1 (quick view of UUID/label)

You need to resize a partition and filesystem. How do you grow a filesystem? What must you do first?
Resize partition first (parted): sudo parted /dev/sdb resizepart 1 100%
Then grow filesystem: sudo resize2fs /dev/sdb1

Why: Must resize partition first, then filesystem. resize2fs for ext2/3/4. Can grow while mounted (if supported), must unmount to shrink.

Alternatives:

  • sudo xfs_growfs /mount/point (grow XFS - operates on mount point)
  • sudo resize2fs /dev/sdb1 50G (resize to specific size)
  • LVM makes resizing easier: lvextend + resize2fs

You want to clone a disk or create a disk image. How do you copy a disk? How do you restore from an image?
Clone disk: sudo dd if=/dev/sda of=/dev/sdb bs=4M status=progress
Create image: sudo dd if=/dev/sda of=disk.img bs=4M status=progress

Why: dd does byte-level copy. if=input file, of=output file, bs=block size. DANGEROUS - wrong of= can wipe disks!

Alternatives:

  • sudo dd if=disk.img of=/dev/sdb bs=4M status=progress (restore)
  • sudo dd if=/dev/sda | gzip > disk.img.gz (compress while imaging)
  • sudo ddrescue /dev/sda /dev/sdb (better for damaged disks)

You want to securely erase a disk before disposal. How do you wipe a disk? What's the difference between quick and secure erase?
Zero-fill disk: sudo dd if=/dev/zero of=/dev/sdb bs=4M status=progress
Secure random wipe: sudo shred -vfz -n 3 /dev/sdb

Why: dd with /dev/zero does one-pass wipe. shred -n 3 does 3 passes with random data then zeros. Good for disposal/donation.

Alternatives:

  • sudo wipefs -a /dev/sdb (wipe signatures only, quick)
  • sudo badblocks -wsv /dev/sdb (destructive write test)
  • SSDs: use ATA Secure Erase instead (dd not effective)

You want to create a swap partition or swap file. How do you set up swap? How do you enable it permanently?
Create swap file: sudo fallocate -l 4G /swapfile && sudo chmod 600 /swapfile
Initialize and enable: sudo mkswap /swapfile && sudo swapon /swapfile

Why: Swap extends RAM to disk. mkswap formats as swap, swapon activates it. Add to /etc/fstab for permanent: /swapfile none swap sw 0 0

Alternatives:

  • sudo mkswap /dev/sdb2 (use partition instead of file)
  • sudo swapoff /swapfile (disable swap)
  • swapon --show or free -h (check swap usage)

You want to check disk health and SMART status. How do you view SMART data? How do you run a disk self-test?
View SMART info: sudo smartctl -a /dev/sda
Run short test: sudo smartctl -t short /dev/sda

Why: SMART (Self-Monitoring, Analysis and Reporting Technology) monitors disk health. Shows temperature, errors, reallocated sectors. Early warning for failure.

Alternatives:

  • sudo smartctl -H /dev/sda (health status only)
  • sudo smartctl -t long /dev/sda (comprehensive test, hours)
  • sudo smartctl -l selftest /dev/sda (view test results)

You want to find the UUID or label of a filesystem. How do you list UUIDs? Why use UUID instead of /dev names?
List all UUIDs: sudo blkid
Specific device: sudo blkid /dev/sdb1

Why: blkid shows UUID, label, filesystem type. UUIDs don't change when devices are added/removed (unlike /dev/sda which can become /dev/sdb). Essential for /etc/fstab.

Alternatives:

  • ls -l /dev/disk/by-uuid/ (see UUID symlinks)
  • ls -l /dev/disk/by-label/ (see label symlinks)
  • lsblk -o NAME,UUID,LABEL (tree view with UUID/label)

You want to benchmark disk performance. How do you test read speed? How do you test write speed?
Test write speed: dd if=/dev/zero of=testfile bs=1G count=1 oflag=direct
Test read speed: dd if=testfile of=/dev/null bs=1G iflag=direct

Why: dd can benchmark disk. oflag=direct bypasses cache for accurate results. Basic test; dedicated tools are better.

Alternatives:

  • hdparm -Tt /dev/sda (quick disk read test)
  • fio (comprehensive I/O testing tool)
  • iozone (filesystem benchmark suite)

You want to create a RAM disk for temporary fast storage. How do you create a tmpfs? What are the use cases?
Create RAM disk: sudo mount -t tmpfs -o size=1G tmpfs /mnt/ramdisk
In fstab: tmpfs /mnt/ramdisk tmpfs size=1G 0 0

Why: tmpfs uses RAM for storage. Extremely fast but data lost on reboot. Good for temp files, caches, build directories.

Use cases:

  • /tmp often uses tmpfs by default
  • Browser cache, compilation temp files
  • Data lost on reboot or unmount!
  • df -h /mnt/ramdisk (check usage)
