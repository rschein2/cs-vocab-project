You created a script file and need to make it executable. How do you add execute permission? What if you want to make it executable for everyone?
Add execute for yourself: chmod +x script.sh
For everyone: chmod a+x script.sh

Why: chmod changes file permissions. +x adds execute permission. Without specifying who, it adds for user/group/others. a+x explicitly means "all".

Alternatives:

  • chmod u+x script.sh (execute for user/owner only)
  • chmod 755 script.sh (rwxr-xr-x: owner full, others read+execute)
  • chmod ug+x script.sh (execute for user and group)

You see permissions like "rwxr-xr--" and want to understand what they mean. How do you read permission notation? What do the three groups represent?
Format: rwxr-xr-- = owner(rwx) group(r-x) others(r--)
Meanings: r=read(4), w=write(2), x=execute(1)

Why: First 3 chars = owner permissions, next 3 = group, last 3 = others. rwx = read+write+execute, r-- = read only, --- = no access.

Examples:

  • rwxr-xr-x = 755 (owner: all, others: read+execute)
  • rw-r--r-- = 644 (owner: read+write, others: read only)
  • rw------- = 600 (owner only, private file)

You want to set exact permissions using numbers instead of letters. How do you use octal notation with chmod? What does each digit mean?
Set exact permissions: chmod 644 file.txt
Calculation: Each digit = sum of r(4) + w(2) + x(1)

Why: Octal notation sets exact permissions. First digit = owner, second = group, third = others. Example: 7=rwx (4+2+1), 6=rw- (4+2), 5=r-x (4+1), 4=r-- (4).

Common patterns:

  • chmod 755 script.sh (rwxr-xr-x: executable script)
  • chmod 644 file.txt (rw-r--r--: regular file)
  • chmod 600 secret.key (rw-------: private file)
  • chmod 700 private_dir/ (rwx------: private directory)

You want new files you create in a directory to have specific default permissions. How do you view your umask? How do you change it?
View current umask: umask
Set umask: umask 022

Why: umask sets default permission mask. It's subtracted from 666 (files) or 777 (directories). umask 022 → files get 644, directories get 755.

Examples:

  • umask 022 → files: 644 (rw-r--r--), dirs: 755 (rwxr-xr-x)
  • umask 077 → files: 600 (rw-------), dirs: 700 (rwx------)
  • umask 002 → files: 664 (rw-rw-r--), dirs: 775 (rwxrwxr-x)
  • Add to ~/.bashrc to make permanent

Set setuid: chmod u+s executable
Numeric notation: chmod 4755 executable

Why: setuid makes executable run as file owner's user ID. Shows as s in owner execute position. Dangerous on shell scripts - can be security risk if misconfigured.
