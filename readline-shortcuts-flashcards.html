Readline/Bash Shortcuts Flashcards

These shortcuts work in: Bash, Zsh, Python REPL, MySQL/PostgreSQL clients, and most CLI tools using GNU Readline

You're typing a long command and need to jump to the beginning of the line. How do you move there instantly? What about jumping to the end?
Beginning: Ctrl+A
End: Ctrl+E

Why: These are the most essential navigation shortcuts. Instead of holding down the left/right arrow keys (slow!), you jump instantly.

Memory trick: Think "A" for the Alphabet start, "E" for End

Related:

  • Home / End keys do the same thing
  • Works in most terminal-based editors and REPLs

You typed a long path and need to fix a typo in the middle. How do you jump forward or backward by whole words instead of one character at a time?
Forward one word: Alt+F (or Esc then F)
Backward one word: Alt+B (or Esc then B)

Why: Much faster than using arrow keys. Especially useful for long commands with multiple arguments or paths.

Example: In git commit -m "fix typo in README.md", you can quickly jump between "git", "commit", "-m", etc.

Note: On macOS, you might need to enable "Use Option as Meta key" in Terminal preferences. Or use Esc then the letter instead.

Single character movement: Ctrl+F (forward) / Ctrl+B (backward)

You've typed most of a command but realize you don't need anything after the cursor. How do you delete everything from the cursor to the end of the line? What about deleting from the cursor to the beginning?
Delete to end of line: Ctrl+K (think: Kill)
Delete to beginning of line: Ctrl+U

Why: These are "kill" commands - they cut the text and store it in the "kill ring" (like a clipboard). You can paste it back with Ctrl+Y (yank).

Example workflow:

$ git commit -m "wrong message here"
# Press Ctrl+U to clear the whole line
$ git status  # type something else

Pro tip: The killed text is saved! You can yank it back with Ctrl+Y if you change your mind.

You need to delete just the previous word (not the whole line). What's the shortcut? What if you want to delete the next word forward?
Delete previous word: Ctrl+W
Delete next word: Alt+D

Why: Perfect for fixing mistakes without clearing the entire command. Deletes from cursor back to the previous space.

Example:

$ git commit -m "typo heer"
# Cursor is at end, press Ctrl+W to delete "heer"
$ git commit -m "here"

Word boundaries: Deletes up to spaces, slashes, or special characters

Difference from Ctrl+U: Ctrl+W deletes one word at a time, Ctrl+U deletes from cursor to line start

You just deleted some text with Ctrl+K or Ctrl+W and want to paste it back. What's the shortcut? Can you paste multiple times?
Paste (yank) deleted text: Ctrl+Y

Why: All the "kill" commands (Ctrl+K, Ctrl+U, Ctrl+W, Alt+D) save text to the "kill ring". Ctrl+Y yanks it back.

Example workflow:

# Copy part of a long command to reuse it
$ docker run --name myapp -p 8080:8080 myimage
# Press Ctrl+A, then Ctrl+K to kill the whole line
$ docker ps  # Check something
$ docker run [Ctrl+Y]  # Yanks back the original command!

Multiple yanks: Yes! Press Ctrl+Y multiple times to paste the same text repeatedly

Think "Y" for Yank - Emacs terminology for paste

You want to clear the terminal screen without typing "clear". What's the keyboard shortcut? Does it preserve your current command?
Clear screen: Ctrl+L

Why: Much faster than typing clear and pressing Enter. Your current typed command stays intact (unlike clear which requires starting over).

What it does:

  • Clears the visible terminal output
  • Preserves whatever you've typed on the current line
  • Keeps your shell history intact

Example: You've typed a long command but the screen is cluttered with output - press Ctrl+L to clean it up while keeping your command.

vs typing "clear": Ctrl+L is instant and doesn't lose what you're typing

You remember running a command recently but can't remember the exact syntax. How do you search backward through your command history interactively? How do you exit the search?
Reverse history search: Ctrl+R
Exit search: Ctrl+G (cancel) or Enter (execute) or Esc (edit)

Why: This is one of the most powerful shortcuts! Instead of pressing Up arrow 50 times, just start typing part of the command.

How it works:

  1. Press Ctrl+R - you'll see (reverse-i-search)
  2. Start typing any part of the command (e.g., "docker")
  3. It shows the most recent match
  4. Press Ctrl+R again to cycle through older matches
  5. Press Enter to run it, or Esc to edit it first

Example: Search for "docker run" - just type Ctrl+R then "docker" and it finds your last docker command

Forward search: Ctrl+S (but often conflicts with terminal flow control)

You just made a mistake editing your command. How do you undo the last change? Can you undo multiple times?
Undo: Ctrl+_ (Ctrl+Shift+Minus) or Ctrl+X Ctrl+U

Why: Just like undo in a text editor! Works for deletions, insertions, and other edits. Can undo multiple times.

What can be undone:

  • Deleted text (Ctrl+W, Ctrl+K, etc.)
  • Typed characters
  • Yanked text (Ctrl+Y)
  • Transpose operations

Example: You accidentally press Ctrl+W too many times - just press Ctrl+_ to undo the deletions one by one.

Keyboard note: On some keyboards, Ctrl+_ is Ctrl+Shift+- (the underscore key)

You typed two characters in the wrong order (like "sl" instead of "ls"). How do you quickly swap them? What about swapping two words?
Transpose characters: Ctrl+T
Transpose words: Alt+T

Why: Common typo fix! Instead of deleting and retyping, just swap them.

Character transpose:

$ sl -la
# Cursor after "l", press Ctrl+T
$ ls -la  # Fixed!

Word transpose:

$ git add commit -m "msg"
# Cursor after "commit", press Alt+T
$ git commit add -m "msg"

Behavior: Swaps character before cursor with character at cursor (or last two chars at end of line)

You want to navigate through your command history one command at a time. What are the keyboard shortcuts? What's the difference from using the Up/Down arrow keys?
Previous command: Ctrl+P (or Up arrow)
Next command: Ctrl+N (or Down arrow)

Why: Ctrl+P/Ctrl+N keep your hands on the home row (more ergonomic for touch typists). Arrow keys work too.

Think: P = Previous, N = Next

Related history shortcuts:

  • !! - Repeat last command (e.g., sudo !!)
  • !$ - Last argument of previous command
  • !^ - First argument of previous command
  • !* - All arguments of previous command
Difference: None functionally! It's personal preference. Use arrow keys or Ctrl+P/N.

You want to repeat the last command. How do you do it quickly? What if you want to run the last command with sudo?
Repeat last command: !!
Repeat with sudo: sudo !!

Why: Incredibly common pattern. You run a command that needs root permissions - just add sudo to the previous command.

Example:

$ apt update
Permission denied
$ sudo !!
# Expands to: sudo apt update

Other bang patterns:

  • !$ - Last argument: mkdir /tmp/test then cd !$ → cd /tmp/test
  • !^ - First argument
  • !* - All arguments
  • !string - Most recent command starting with "string"
Safety tip: Use :p to preview first: sudo !!:p shows what would run

You want to reuse the last argument from the previous command. How do you insert it without retyping? What about using the first argument instead?
Last argument: !$ or Alt+. (Alt+period)
First argument: !^

Why: Super common workflow! You just created a directory and want to cd into it, or operated on a file and want to edit it.

Examples:

$ mkdir /var/www/myapp
$ cd !$
# Same as: cd /var/www/myapp

$ cat /etc/nginx/nginx.conf
$ sudo vim !$
# Same as: sudo vim /etc/nginx/nginx.conf

Alt+. is even better: Press repeatedly to cycle through last arguments of previous commands!

All arguments: !* inserts all arguments from previous command

You started typing a command but realize you need to run something else first. How do you temporarily save the current line, run another command, then get it back?
Save and clear line: Ctrl+U (kill line), run other command, then Ctrl+Y (yank back)

Why: Lets you "park" a command while you check something, without losing your work.

Better method:

# Type a long command
$ docker run --name web -p 8080:8080 --env-file .env myimage
# Ctrl+U to clear and save it
$ docker ps  # Check running containers
$ [Ctrl+Y]  # Paste it back!
$ docker run --name web -p 8080:8080 --env-file .env myimage

Alternative: Use a comment!

# Add # to start, press Enter, it's saved in history
$ #docker run --name web -p 8080:8080 myimage
$ docker ps
$ [Up arrow twice] # Get it back, delete the #

You want to stop the current command from running without closing the terminal. What's the shortcut? What about suspending a running program to resume later?
Interrupt/kill current command: Ctrl+C
Suspend (pause): Ctrl+Z

Why: Ctrl+C sends SIGINT to terminate the process. Ctrl+Z sends SIGTSTP to suspend it (you can resume with fg or run in background with bg).

Example - Suspend workflow:

$ vim myfile.txt
# Press Ctrl+Z to suspend vim
[1]+ Stopped    vim myfile.txt
$ ls  # Do something else
$ fg  # Resume vim (brings it to foreground)

Related:

  • Ctrl+D - Send EOF (end of input), closes shell if line is empty
  • fg - Resume suspended job in foreground
  • bg - Resume suspended job in background
  • jobs - List suspended/background jobs

You're in the middle of typing a command but want to quickly open it in your default editor ($EDITOR) to edit it more comfortably. What's the shortcut?
Edit command in $EDITOR: Ctrl+X Ctrl+E

Why: For very long or complex commands (like multi-line scripts), it's easier to edit in vim/nano/emacs with full editing capabilities.

Workflow:

  1. Start typing a long command
  2. Press Ctrl+X Ctrl+E
  3. Your editor opens with the command
  4. Edit it, save, and quit
  5. Command runs automatically (or appears in your prompt if you cancel)

Example use case:

# Typing a complex find/xargs command
$ find . -name "*.log" -mtime +30 -exec ...
# Ctrl+X Ctrl+E opens in vim/nano to edit comfortably
Editor: Uses $EDITOR or $VISUAL environment variable (usually vim, nano, emacs)

You want to insert a comment in your command line that gets saved in history but doesn't execute. Why would you do this? How?
Comment trick: Start line with #, or press Alt+# to insert # at the beginning

Why: Save a command for later without executing it! Great for complex commands you're building or want to remember.

Example:

$ #TODO: docker run --name web -p 8080:8080 myimage

$ history | tail -1

# Later, press Ctrl+R and search for "TODO"

Alt+# shortcut: Automatically adds # to the beginning of your current line and executes it (saves to history)

Use cases:

  • Save a complex command you're building
  • Document what you were trying to do
  • Create command templates to reuse later

You want to see all available keyboard shortcuts in your current shell. What command shows you a complete list? How do you search for a specific key binding?
List all key bindings: bind -P (Bash) or bindkey (Zsh)
Show specific binding: bind -p | grep 'command-name'

Why: Readline has dozens of shortcuts! This shows everything available and what they do.

Example output:

$ bind -P | grep '^beginning-of-line'
beginning-of-line can be invoked via "\C-a", "\eOH", "\e[1~", "\e[H".

Useful searches:

  • bind -P | grep kill - All kill commands
  • bind -P | grep yank - Paste commands
  • bind -P | grep history - History shortcuts

Custom bindings: Add to ~/.inputrc to customize or add new shortcuts

See what a key does: bind -q command-name shows which keys trigger it

You want to customize Readline shortcuts or behavior (like case-insensitive tab completion). What configuration file do you edit? Do the changes apply to all Readline programs?
Config file: ~/.inputrc

Why: This file controls Readline behavior for ALL programs that use it: Bash, Python REPL, MySQL client, etc.

Useful settings:

# Case-insensitive tab completion
set completion-ignore-case on

# Show all completions immediately (no bell)
set show-all-if-ambiguous on

# Use visible bell instead of beep
set bell-style visible

# Vi mode instead of Emacs (advanced)
set editing-mode vi

Custom key bindings:

# Make Ctrl+/ undo (easier than Ctrl+_)
"\C-/": undo

# Alt+h deletes previous word (alternative to Ctrl+W)
"\eh": backward-kill-word

Reload config: bind -f ~/.inputrc or restart terminal

Applies to: Any program using GNU Readline library

Your terminal is showing garbled characters or seems frozen. What shortcuts can help? How do you reset the terminal display?
Unfreeze terminal: Ctrl+Q
Reset terminal: reset or Ctrl+L then type stty sane

Why this happens: You might have accidentally pressed Ctrl+S (freezes output) or displayed binary data (garbles the terminal).

Common scenarios:

1. Frozen terminal (can't type):

  • Cause: Accidentally pressed Ctrl+S (XOFF flow control)
  • Fix: Press Ctrl+Q (XON) to unfreeze

2. Garbled display:

  • Cause: Displayed binary file (like cat image.png)
  • Fix: Type reset (blindly if needed) or stty sane

Prevention: Disable flow control: add stty -ixon to ~/.bashrc

Quick clean: Close and reopen the terminal tab if reset doesn't work

How do tab completion and these shortcuts interact? What happens if you try to complete a word and there are multiple matches? How do you cycle through completions?
Tab completion: Tab (complete) or Tab Tab (show all)
Cycle through matches: Alt+* (insert all) or configure menu-complete

Default behavior:

  • Tab once: completes if unique, or completes common prefix
  • Tab twice: shows all possible matches

Example:

$ cd Doc[Tab]
# If only "Documents/" exists → completes to Documents/
# If "Documents/" and "Downloads/" exist → shows both

$ git ch[Tab][Tab]
checkout  cherry  cherry-pick  check-attr

Advanced - Menu completion (cycles through):

# Add to ~/.inputrc
TAB: menu-complete
"\e[Z": menu-complete-backward  # Shift+Tab goes backward

Insert all matches: Alt+* inserts all completions at once

Completion and editing: After tab completion, use Ctrl+W/K/U to edit the completed text

What's the difference between Emacs mode and Vi mode in Readline? How do you switch between them? Which one are these shortcuts for?
Default mode: Emacs (these shortcuts)
Switch to Vi mode: set -o vi or in ~/.inputrc: set editing-mode vi
Switch back to Emacs: set -o emacs

Emacs mode (default):

  • All the shortcuts in this deck (Ctrl+A, Ctrl+E, Ctrl+K, etc.)
  • Always in "insert" mode
  • More intuitive for most users

Vi mode:

  • Modal editing like Vim (insert mode vs normal mode)
  • Press Esc to enter normal mode, then use h/j/k/l, w/b, dd, etc.
  • Great if you're a heavy Vim user

Check current mode: set -o | grep emacs

Learning curve: Emacs mode is easier to start with; Vi mode is powerful if you know Vim
