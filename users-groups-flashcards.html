You need to create a new user account on the system. How do you add a user? What if you want to set their home directory and shell?
Add user (interactive): sudo adduser username
Add with options: sudo useradd -m -s /bin/bash username

Why: adduser is user-friendly (Debian/Ubuntu), creates home dir, prompts for password. useradd is low-level. -m creates home, -s sets shell.

Alternatives:

  • sudo useradd -m -d /home/custom -s /bin/zsh username (custom home)
  • sudo useradd -m -G sudo username (add to sudo group)
  • sudo passwd username (set password after creation)

You need to delete a user account. How do you remove a user? What if you want to delete their home directory too?
Remove user: sudo userdel username
Remove with home directory: sudo userdel -r username

Why: userdel removes user account. -r removes home directory and mail spool. Without -r, files remain (can be useful for backups).

Alternatives:

  • sudo deluser username (Debian/Ubuntu friendly wrapper)
  • sudo deluser --remove-home username (remove home)
  • Check for leftover files: find / -user username 2>/dev/null

Change shell: sudo usermod -s /bin/zsh username
Change home directory: sudo usermod -d /new/home -m username

Why: usermod modifies user accounts. -s changes login shell. -d changes home directory, -m moves contents to new location.

Set password: sudo passwd username
Force change on login: sudo passwd -e username

Why: passwd changes passwords. -e expires password immediately, forcing change on next login. Users can change their own with just passwd.

You need to view and edit the user database files directly. Where are users stored? Where are passwords hashed?
User database: /etc/passwd (user info)
Password hashes: /etc/shadow (requires root)

Why: /etc/passwd contains username, UID, GID, home, shell (world-readable). /etc/shadow has password hashes and aging info (root only, for security).

File format:

  • /etc/passwd: user:x:UID:GID:comment:home:shell
  • /etc/shadow: user:$hash:lastchange:min:max:warn:...
  • vipw (safely edit /etc/passwd)
  • vipw -s (safely edit /etc/shadow)

You want to create a system user for running a service (no login, no home). How do you create a system user? What makes it different?
Create system user: sudo useradd -r -s /usr/sbin/nologin servicename
With home in /var: sudo useradd -r -d /var/lib/servicename -s /usr/sbin/nologin servicename

Why: -r creates system user (UID < 1000). /usr/sbin/nologin prevents shell login. System users run services/daemons securely without human access.

Alternatives:

  • sudo useradd -r -M servicename (-M = no home directory)
  • sudo adduser --system --no-create-home servicename (Debian)
  • Common shells: /bin/false, /usr/sbin/nologin

You want to lock a user account temporarily without deleting it. How do you lock an account? How do you unlock it?
Lock account: sudo usermod -L username or sudo passwd -l username
Unlock account: sudo usermod -U username or sudo passwd -u username

Why: Locking puts ! in password hash, preventing password login. SSH keys still work! User keeps files and processes.

Alternatives:

  • sudo usermod -s /usr/sbin/nologin username (disable shell completely)
  • sudo usermod -e 1 username (expire account)
  • To fully disable: lock + change shell + expire

You want to create multiple users with similar settings efficiently. How do you script user creation? What's a safe approach?
Script approach: 
while IFS=, read user fullname; do
  sudo useradd -m -c "$fullname" "$user"
  sudo passwd -e "$user"
done < users.csv

Why: Loop through CSV file, create users with full names, force password change. -c sets comment (full name). Safer than batch with passwords in file.

Alternatives:

  • newusers command (bulk user creation from file)
  • Use Ansible/Puppet for production environments
  • Generate random passwords: pwgen or openssl rand

You need to audit user accounts for security. How do you find users with UID 0? How do you find users with empty passwords?
Find UID 0 (root-equivalent): awk -F: '$3 == 0 {print $1}' /etc/passwd
Find empty passwords: sudo awk -F: '$2 == "" {print $1}' /etc/shadow

Why: UID 0 = root privileges. Only "root" should have UID 0. Empty password = security risk. Regular audits are important.

Other security checks:

  • sudo awk -F: '$3 < 1000 && $1 != "root" {print $1}' /etc/passwd (system users)
  • find / -nouser -o -nogroup (files with no valid owner)
  • last | grep -v 'still logged in' | head -20 (recent logins)
