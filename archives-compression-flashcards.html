You need to create a tarball archive of a directory for backup. How do you create a tar archive? What if you want to compress it with gzip at the same time?
Create tar archive: tar -cf archive.tar directory/
Create and gzip: tar -czf archive.tar.gz directory/

Why: tar -c creates archive, -f specifies filename. -z adds gzip compression. Convention: .tar for uncompressed, .tar.gz or .tgz for gzipped.

Alternatives:

  • tar -cjf archive.tar.bz2 directory/ (bzip2 compression, better ratio)
  • tar -cJf archive.tar.xz directory/ (xz compression, best ratio)
  • tar -cvzf archive.tar.gz dir/ (-v for verbose output)

You downloaded a .tar.gz file and need to extract it. How do you extract a tar.gz archive? What if you want to see what's inside before extracting?
Extract tar.gz: tar -xzf archive.tar.gz
List contents first: tar -tzf archive.tar.gz

Why: -x extracts, -z handles gzip, -f specifies file. -t lists contents without extracting. Good to check before extracting.

Alternatives:

  • tar -xzf archive.tar.gz -C /destination/ (extract to specific directory)
  • tar -xzvf archive.tar.gz (-v shows files as extracted)
  • tar -xzf archive.tar.gz file.txt (extract only specific file)

You want to add files to an existing tar archive or update files that have changed. How do you append to a tar? How do you update modified files?
Append to archive: tar -rf archive.tar newfile.txt
Update changed files: tar -uf archive.tar directory/

Why: -r appends files to end of archive. -u only adds files newer than version in archive. Note: can't append to compressed (.gz, .bz2) archives!

Alternatives:

  • For compressed archives: extract, add files, re-compress
  • tar -rvf archive.tar file1 file2 (append multiple files)
  • tar --delete -f archive.tar unwanted.txt (remove from archive)

You need to compress a single file to save space. How do you gzip a file? How do you decompress it later?
Compress file: gzip file.txt (creates file.txt.gz, removes original)
Decompress: gunzip file.txt.gz or gzip -d file.txt.gz

Why: gzip compresses and replaces original file. gunzip decompresses and removes .gz. Fast compression, widely supported.

Alternatives:

  • gzip -k file.txt (keep original file after compressing)
  • gzip -9 file.txt (maximum compression, slower)
  • gzip -1 file.txt (fast compression, larger file)
  • zcat file.txt.gz (view without decompressing)

You want better compression than gzip. How do you use bzip2? How does it compare to gzip?
Compress with bzip2: bzip2 file.txt (creates file.txt.bz2)
Decompress: bunzip2 file.txt.bz2 or bzip2 -d file.txt.bz2

Why: bzip2 offers better compression than gzip but slower. Good for large files when size matters more than speed. Convention: .bz2 extension.

Alternatives:

  • bzip2 -k file.txt (keep original)
  • bzcat file.txt.bz2 (view without decompressing)
  • tar -cjf archive.tar.bz2 dir/ (tar with bzip2)
  • bzip2 typically 10-15% smaller than gzip, 2-3x slower

You want the best possible compression ratio. How do you use xz compression? What are the tradeoffs?
Compress with xz: xz file.txt (creates file.txt.xz)
Decompress: unxz file.txt.xz or xz -d file.txt.xz

Why: xz provides best compression ratio but slowest speed and high memory usage. Great for distributing software. Uses LZMA2 algorithm.

Alternatives:

  • xz -k file.txt (keep original)
  • xz -9 file.txt (maximum compression, very slow)
  • tar -cJf archive.tar.xz dir/ (tar with xz)
  • xzcat file.txt.xz (view without decompressing)

You need to create a zip archive for Windows compatibility. How do you create a zip file? How do you extract one?
Create zip: zip archive.zip file1.txt file2.txt
Extract zip: unzip archive.zip

Why: zip creates archives compatible with Windows. Unlike gzip, zip is both archiver and compressor (like tar+gzip combined). Widely compatible.

Alternatives:

  • zip -r archive.zip directory/ (recursive: include subdirectories)
  • unzip -l archive.zip (list contents without extracting)
  • unzip archive.zip -d /destination/ (extract to specific directory)
  • zip -9 archive.zip file.txt (maximum compression)

You want to add or update files in an existing zip archive. How do you add to a zip? How do you delete files from it?
Add/update files: zip archive.zip newfile.txt
Delete from archive: zip -d archive.zip unwanted.txt

Why: zip can modify existing archives. If file exists, it updates; if new, it adds. -d deletes files from archive.

Alternatives:

  • zip -u archive.zip file.txt (-u only updates if newer)
  • zip -r archive.zip newdir/ (add entire directory)
  • unzip -p archive.zip file.txt (extract to stdout, not file)

You need to create a password-protected zip archive. How do you encrypt a zip? How do you extract it?
Create encrypted zip: zip -e archive.zip file.txt (prompts for password)
Extract: unzip archive.zip (prompts for password)

Why: -e encrypts zip with password. Uses standard zip encryption (weak) or AES (stronger with -P on some systems). Good for basic protection.

Alternatives:

  • zip -er archive.zip directory/ (recursive + encrypted)

Create 7z archive: 7z a archive.7z directory/
Extract 7z: 7z x archive.7z

View gzipped file: zcat file.txt.gz or zless file.txt.gz
Search in gzipped file: zgrep "pattern" file.txt.gz

Why: zcat decompresses to stdout, zless allows paging. zgrep searches without extracting. Saves disk space and time.

  • zless file.txt.gz (paginated viewing like less)

You want to compare the compression ratios of different algorithms. How do you test compression effectiveness? What factors affect it?
Test gzip: gzip -c file > file.gz && ls -lh file file.gz
Compare multiple: Test with gzip, bzip2, xz and check sizes

Why: -c outputs to stdout (preserves original). ls -lh shows human-readable sizes. Compression depends on file type and content.

Typical ratios (text):

  • gzip: 60-70% reduction, fast (good default)
  • bzip2: 65-75% reduction, slower
  • xz: 70-80% reduction, slowest, high memory
  • Already-compressed (images, videos): minimal benefit

Extract specific file: tar -xzf archive.tar.gz path/to/file.txt

You want to create an archive but exclude certain files or directories. How do you exclude files from tar? What if you have a long list to exclude?
Exclude pattern: tar -czf archive.tar.gz --exclude='*.log' directory/
Exclude multiple: tar -czf archive.tar.gz --exclude='*.log' --exclude='tmp/' directory/

Why: --exclude skips matching files/directories. Can use multiple times or patterns. Useful for avoiding temp files, logs, node_modules, etc.

You need to split a large archive into smaller chunks for transfer. How do you split archives? How do you reassemble them?
Create and split: tar -czf - directory/ | split -b 100M - archive.tar.gz.
Reassemble and extract: cat archive.tar.gz.* | tar -xzf -

Why: tar -czf - outputs to stdout, split -b creates chunks (100M each). cat concatenates them back, pipe to tar -xzf - to extract.

Alternatives:

  • split -d -b 100M archive.tar.gz chunk_ (split existing archive)
  • cat chunk_* > archive.tar.gz && tar -xzf archive.tar.gz
  • 7z a -v100m archive.7z directory/ (7zip built-in split)

Preserve metadata (default): tar -czf archive.tar.gz directory/
Restore with permissions: sudo tar -xzpf archive.tar.gz

Why: tar preserves permissions/timestamps by default. -p on extraction preserves exact permissions (needs sudo for ownership). Useful for backups.

You need to verify the integrity of an archive before extracting. How do you test archives? What if the archive is corrupted?
Test tar.gz integrity: gunzip -t archive.tar.gz
Test zip integrity: unzip -t archive.zip

Why: -t tests archive integrity without extracting. Catches corruption early. For tar.gz, test compression first, then tar if needed.

  • 7z t archive.7z (test 7z archive)

You want to create incremental backups with tar. How do you create a snapshot? How do you create incremental based on changes?
Full backup with snapshot: tar -czf full.tar.gz -g snapshot.file directory/
Incremental backup: tar -czf incremental.tar.gz -g snapshot.file directory/

Why: -g creates/uses snapshot file to track changes. First run = full backup. Subsequent runs = only changed files. Snapshot file maintains state.

Alternatives:

  • tar -czf backup-$(date +%Y%m%d).tar.gz dir/ (dated backups)
  • rsync -a --link-dest=../previous/ src/ dest/ (rsync incremental)
  • Professional tools: duplicity, restic, borg

You need to extract an archive but preserve the directory structure without creating a top-level directory. How do you strip leading components? How do you flatten the structure?
Strip 1 directory level: tar -xzf archive.tar.gz --strip-components=1
Flatten to current dir: tar -xzf archive.tar.gz --transform='s/.*\///'

Why: --strip-components=N removes N leading directories from paths. --transform uses sed to modify paths. Useful when archive has unwanted wrapper directory.

Alternatives:

  • tar -xzf archive.tar.gz -C tempdir/ && mv tempdir/*/* .
  • unzip -j archive.zip (-j junks paths, extracts flat)
