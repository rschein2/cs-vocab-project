You want to create a shortcut to a file in another location. How do you create a symbolic link? What if you want to link a directory?
Create symlink: ln -s /path/to/original /path/to/link
Link directory: ln -s /path/to/directory /path/to/link

Why: ln -s creates symbolic link (symlink). Symlinks are shortcuts that point to another file/directory. Works across filesystems. Like Windows shortcuts.

Alternatives:

  • ln -sf target link (force: overwrite existing link)
  • ln -sr /path/file link (relative symlink)
  • Use absolute paths to avoid broken links

You want to see where a symbolic link points to. How do you read a symlink? How do you resolve all symlinks in a path?
Read symlink: readlink linkname
Resolve to absolute path: readlink -f linkname

Why: readlink shows what a symlink points to. -f follows all symlinks and returns absolute path to final target. Useful for debugging.

Alternatives:

  • ls -l linkname (shows: linkname -> target)
  • realpath linkname (canonical absolute path)
  • file linkname (shows if it's a symlink and target)

You want to understand the difference between symbolic and hard links. What's a hard link? When would you use it instead of a symlink?
Create hard link: ln /path/to/original /path/to/hardlink
Key difference: Hard link = same file, different name. Symlink = pointer to file.

Why: Hard links share inode with original - same file in two places. Can't cross filesystems or link directories. File exists until all hard links deleted.

Comparison:

  • Hard link: survives if original deleted, same filesystem only
  • Symlink: breaks if original deleted, works across filesystems
  • ls -li (shows inode numbers - same for hard links)
  • Hard links don't work for directories (prevents loops)

You want to find all symlinks in a directory or find broken symlinks. How do you find symlinks? How do you find broken ones?
Find all symlinks: find /path -type l
Find broken symlinks: find /path -type l -xtype l

Why: -type l finds symbolic links. -xtype l finds links whose target doesn't exist (broken links). Useful for cleanup.

Alternatives:

  • find /path -type l -ls (list details of symlinks)
  • find /path -type l -exec readlink -f {} \; (show targets)
  • find /path -type l ! -exec test -e {} \; -print (broken links)

You want to find files that are hard-linked together. How do you find hard links to a file? How many hard links does a file have?
Check link count: ls -l file.txt (2nd column shows count)
Find all hard links: find / -samefile file.txt 2>/dev/null

Why: ls -l shows hard link count (usually 1). find -samefile finds all paths pointing to same inode. Useful for duplicate detection.

Alternatives:

  • ls -li file.txt (show inode number)
  • find / -inum 123456 (find by inode number)
  • stat file.txt (detailed file info including links)

Make immutable: sudo chattr +i filename

Why: chattr +i makes file unchangeable - can't modify, delete, or rename even as root. lsattr lists extended attributes. Extreme protection.

You want to make a log file append-only so old entries can't be deleted. How do you set append-only attribute? What other useful attributes exist?
Append-only: sudo chattr +a logfile.txt
No atime updates: sudo chattr +A file.txt

Why: +a allows appending but not modification or deletion. Good for logs. +A disables access time updates (performance boost).

Common attributes:

  • i = immutable (unchangeable)
  • a = append only
  • A = no atime updates
  • c = compressed automatically
  • d = no dump (excluded from backup)

You want to create a relative symlink that works even if you move the directory. How do you create relative links? What's the advantage?
Relative symlink: ln -s ../target linkname
Auto-relative: ln -sr /abs/path/to/target linkname

Why: Relative symlinks use relative paths (../file). If you move the directory, links still work. -r automatically makes link relative.

Example:

  • Absolute: link -> /home/user/file (breaks if moved)
  • Relative: link -> ../file (survives moves)
  • Use relative for portable directory structures
  • readlink link (shows relative path)

You want to replace a directory with a symlink to move data without breaking paths. How do you safely replace? What if the link already exists?
Replace directory with link: 
mv /old/location /new/location
ln -s /new/location /old/location
Force replace existing link: ln -sf /new/target /old/link

Why: Move data first, then create symlink at old location. Applications using old path still work. -f force-replaces existing link.

Alternatives:

  • ln -sfn /new/target /old/link (-n treats link to dir as file)
  • Test first: ls -l /old/location
  • Common pattern for data migration

You want to update file access or modification time without editing. How do you change timestamps? How do you match another file's timestamp?
Update to now: touch filename
Match another file's time: touch -r reference.txt target.txt

Why: touch updates access and modify times to current time. Creates file if doesn't exist. -r copies timestamp from reference file.

Alternatives:

  • touch -a file (only access time)

You want to understand the three different timestamps on files. What are atime, mtime, and ctime? How do they differ?
View all timestamps: stat filename
Timestamps: atime (access), mtime (modify), ctime (change)

Why: atime = last read, mtime = content modified, ctime = metadata changed (permissions, ownership). ctime can't be manually set.

Differences:

  • atime: reading file updates it (often disabled for performance)
  • mtime: editing file content updates it
  • ctime: chmod, chown, or content change updates it
  • ls -l shows mtime, ls -lc shows ctime, ls -lu shows atime

You want to find files modified in the last 24 hours or accessed recently. How do you find files by modification time? How about by access time?
Modified in last 24hrs: find /path -mtime -1
Accessed in last 7 days: find /path -atime -7

Why: -mtime -1 = modified less than 1 day ago. -atime -7 = accessed less than 7 days ago. Useful for finding recent changes.

Alternatives:

  • find /path -mmin -60 (modified in last 60 minutes)
  • find /path -mtime +30 (modified MORE than 30 days ago)
  • find /path -ctime -1 (metadata changed in last day)
  • find /path -newer reference.txt (modified after reference)

You want to preserve symlinks when copying or find the inode number of a file. How do you copy preserving symlinks? How do you see inodes?
Copy preserving symlinks: cp -a source dest
Show inode numbers: ls -li

Why: cp -a (archive mode) preserves symlinks, permissions, timestamps. Without it, cp dereferences symlinks. ls -i shows inode numbers.

Alternatives:

  • cp -P link newlink (copy symlink itself, not target)
  • cp -L link newfile (dereference: copy target, not link)
  • stat file (detailed inode info)

You want to create multiple symlinks at once or update existing symlinks. How do you create symlinks efficiently? How do you update broken links?
Create multiple symlinks: 
for file in /source/*.txt; do
  ln -s "$file" /dest/
done
Update all symlinks in directory: find /path -type l -delete && create new links

Why: Loop through files to create symlinks. For updating, find and delete broken links, then recreate. -f can replace individual links.

Alternatives:

  • ln -sf /new/target /old/link (update single link)
  • find /path -type l -xtype l -delete (delete broken links only)
  • symlinks -r /path (symlinks tool: convert absolute to relative)
